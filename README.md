# N-Body Orbital Dynamics Sandbox

## Идея проекта
Проект представляет собой высокопроизводительный симулятор гравитационного взаимодействия N тел в 2D-пространстве. Основная цель — не просто визуализация, а создание эффективного вычислительного движка, способного обрабатывать большое количество объектов (звезд, планет, астероидов) с использованием продвинутых алгоритмов оптимизации сложности вычислений.

## Запланированные фичи по итерациям

### Итерация 1 (Дедлайн: 26.04)
1. **Core Physics Engine:** Реализация физического движка с поддержкой взаимного притяжения тел ($F = G \frac{m_1 m_2}{r^2}$). Использование численных методов интегрирования (например, метод Верле для лучшей сохранности энергии системы).
2. **Entity-Component-System (ECS) Architecture:** Логическое разделение данных тел (масса, позиция) и систем, которые ими управляют (гравитация, коллизии).
3. **Interactive UI (PyQt6):** Главное окно симуляции, панель управления параметрами (гравитационная постоянная, масштаб) и функционал создания тел "броском" мыши (drag-to-launch).

### Итерация 2 (Дедлайн: 10.05)
1. **Алгоритм Барнса-Хата (Barnes-Hut):** Оптимизация вычисления гравитации с использованием Quadtree (четверичного дерева), снижающая сложность с $O(N^2)$ до $O(N \log N)$.
2. **Система коллизий и слияний:** Обработка столкновений тел с сохранением импульса и изменением массы/радиуса.
3. **Профилирование и оптимизация:** Сравнение производительности наивного алгоритма и Барнса-Хата. Анализ узких мест через `cProfile` и `line_profiler`, внедрение `numpy` для векторизации расчетов.
